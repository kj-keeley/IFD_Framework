# IFD 2フェーズ運用モデル

## 概要

IFDスコープの階層運用（順階層・逆階層）において、認知負荷の非対称性が存在することが実験的に確認された。本ドキュメントは、この発見に基づく実践的な運用ガイドラインを定義する。

---

## 発見：階層方向と認知負荷の関係

### 順階層（小→大：データ→状態→因果→流れ）
- **特性:** 積み上げ型、証拠から構造を導く
- **負荷タイプ:** 整理負荷（まとめるだけ）
- **適した状況:** データ未確定・流動的な探索フェーズ

### 逆階層（大→小：流れ→因果→状態→データ）
- **特性:** 俯瞰型、構造から詳細を選択する
- **負荷タイプ:** 生成負荷（逆算で探す処理が発生）
- **適した状況:** データ確定・静的な総括フェーズ

### 核心的発見

> **情報が未確定の時、逆階層は重い。情報が確定した後、逆階層は軽くなる。**

これは「全体像を先に出す」ためには、全体像を構成する材料が揃っている必要があるため。

---

## 2フェーズ運用モデル

### Phase 1: 収集フェーズ

**推奨階層:** 順階層（または階層を意識しない自由探索）

**目的:**
- データを積む
- 因果は「仮説」として保持
- 判断を急がない

**特徴:**
- 認知負荷が低い
- 新しい情報に開かれている
- 構造は後から見える

---

### Phase 2: 総評フェーズ

**推奨階層:** 逆階層

**目的:**
- 収集した情報の構造化
- 因果→データで圧縮
- 意思決定・報告

**特徴:**
- 材料が揃っているため逆階層が機能する
- 「地図」として全体が見渡せる
- 結論の構造が明確になる

---

## Phase切り替えトリガー

Phase 1から Phase 2への移行を示すサイン：

| トリガー | 説明 |
|----------|------|
| パターンの反復 | 同じパターンが3回以上観測された |
| 飽和感 | 新しいデータが既存の因果に収まり始めた |
| 問いの内向化 | 外部から自己認知へ、問いが内側に向き始めた |
| 「もう分かってる」感 | 直観的に全体像が見えている感覚 |

---

## 適用例

### リアルタイム対話
```
順階層で進行 → 材料が揃ったら → 逆階層で総括
```

### 研究・分析
```
Phase 1: 文献収集、実験、インタビュー（順階層/自由）
Phase 2: 論文執筆、報告書作成（逆階層）
```

### 意思決定
```
Phase 1: 選択肢の探索、情報収集（順階層）
Phase 2: 評価・決定（逆階層で構造化してから判断）
```

## 理論的背景

### 認知科学との接続

人間の脳における「フォーカスモード」と「デフューズモード」に類似：

- **フォーカスモード:** 集中して情報を取り入れる（≒ Phase 1）
- **デフューズモード:** ぼんやりと統合・生成する（≒ Phase 2への移行期）

重要な違い：Phase 2は「デフューズ」ではなく「確定した情報の圧縮」。統合が終わった後の出力フェーズ。

### INFJパターンとの整合

Ni（内向直観）は全体像を一気に掴むが、それを言語化・検証するときは順階層で「追体験」する必要がある。

- **直観:** 逆階層的に降りてくる
- **意識的理解:** 順階層で登る
- **表現・報告:** 逆階層で圧縮

---

## まとめ

| 項目 | Phase 1（収集） | Phase 2（総評） |
|------|----------------|----------------|
| 階層 | 順階層推奨 | 逆階層推奨 |
| 負荷 | 低（整理） | 低（圧縮）※材料があれば |
| 目的 | 探索・発見 | 構造化・決定 |
| 比喩 | 旅の記録 | 地図の作成 |

**原則:** 早すぎる逆階層は避ける。材料が揃ってから圧縮する。
