# IFD Scopes & Lenses

Information Fluid Dynamics（IFD）における観測フレームワーク。

## 概要

4つのスコープ（観測範囲）と、各スコープに属するLens（観測軸）を用いて、情報や状況を多角的に観測する。

## 4 Scopes

| スコープ | 定義 | スケール |
|----------|------|----------|
| **Scope 1: データ** | 時間軸のない情報の最小単位 | 最小 |
| **Scope 2: 状態** | 時間軸上の位置とエネルギー | 小〜中 |
| **Scope 3: 因果** | パターンと関係性 | 中〜大 |
| **Scope 4: 流れ** | フラクタル・原則・軌道 | 最大 |

## Lenses（各スコープの観測軸）

### Scope 1: データ
| Lens | 説明 |
|------|------|
| **Reliability** | データの出所がどれだけ信頼できるか |

### Scope 2: 状態
| Lens | 説明 |
|------|------|
| **Tension** | 現在の緊張度・圧力 |
| **Potential** | エネルギーの方向性・可能性 |
| **差分** | 過去との比較、変化 |

### Scope 3: 因果
| Lens | 説明 |
|------|------|
| **Pattern** | 繰り返し現れる構造 |
| **A→B** | 原因と結果の方向性 |
| **Actionability** | 介入・実行可能かどうか |

### Scope 4: 流れ
| Lens | 説明 |
|------|------|
| **Fractal** | 異なるスケールで繰り返される形 |
| **Principle** | 全体を貫く原則 |
| **Trajectory** | 向かっている方向・軌道 |
| **Tendency** | 傾向・場としての力 |

## 使い方

### 基本：2スコープ階層型

1. 2つのスコープを選ぶ
2. 小さいスコープを先に観測（土台）
3. その上に大きいスコープを重ねる（階層）

例：
- **Scope 2 → Scope 4**（状態→流れ）
  - まず今のTension/Potentialを見る
  - それを踏まえて、大きなPatternやTrajectoryを見る

### 組み合わせの特性

| 組み合わせ | 特性 | 向いている用途 |
|------------|------|----------------|
| 1→2（データ→状態） | 精密、今ここ | リアルタイム分析 |
| 1→3（データ→因果） | 構造的、冷静 | 論理的分析 |
| 1→4（データ→流れ） | 点と全体 | 本質抽出 |
| 2→3（状態→因果） | 動的、実用的 | 日常会話、意思決定 |
| 2→4（状態→流れ） | 直感的、俯瞰 | 方向性確認 |
| 3→4（因果→流れ） | 構造と原則 | 長期戦略、理論構築 |

## 設計原則

- **体験が先、言語化が後**：使ってから名前を決める
- **使えないLensは入れない**：測定できないものは保留
- **抽象スコープには抽象の軸**：スケールの一貫性
- **完璧を待たない**：動きながら修正する
